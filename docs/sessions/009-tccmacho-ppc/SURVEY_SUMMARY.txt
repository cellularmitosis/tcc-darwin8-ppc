 MAJOR FINDINGS
================================================================================

1. STRUCTURE DIFFERENCES (32-bit vs 64-bit Mach-O)
   - mach_header:        28 bytes (vs 64-bit: 32 bytes) — save 4 bytes
   - segment_command:    48 bytes (vs 64-bit: 72 bytes) — save 24 bytes
   - section:            48 bytes (vs 64-bit: 80 bytes) — save 32 bytes
   - nlist:              12 bytes (vs 64-bit: 16 bytes) — save 4 bytes
   
   STRATEGY: Use per-arch typedefs (Mach_Header, Segment_Command, Section_Mach)
             to minimize code duplication and reduce #ifdef clutter.

2. LOAD COMMAND INCOMPATIBILITIES (Tiger 10.4 Constraints)
   - LC_MAIN is 10.7+ only (NOT available on Tiger)
     → Use LC_UNIXTHREAD (0x5) instead for PPC executables
     → Requires thread state structure (40 uint32_t for PPC)
   
   - LC_BUILD_VERSION is 10.6+ only (NOT available on Tiger)
     → Omit entirely for PPC
   
   - LC_DYLD_INFO_ONLY (10.6+) vs LC_DYLD_CHAINED_FIXUPS (10.15+)
     → Use classic REBASE/BINDING sections (already in code as #else branch)

3. ENDIANNESS (Big-Endian for PPC)
   - write32le() used throughout for x86_64/arm64 (little-endian)
   - PPC is big-endian → need write32be() equivalent
   
   SOLUTION: Define conditional write32le macro:
     #ifdef TCC_TARGET_PPC
         #define write32le(p, v) write32be(p, v)
     #endif
   
   Minimal changes required (one helper definition, used everywhere).

4. INSTRUCTION SEQUENCES (8 Code Blocks Requiring PPC)
   
   a) __GLOBAL_init (lines 575-645)
      - Calling convention: args in r3-r6, save LR, frame setup
      - PPC sequence: lis/ori (load addr), bl/mtlr (calls)
      - Relocations: R_PPC_HA16, R_PPC_LO16, R_PPC_REL24
   
   b) PLT Stub Generation (2 locations: lines 735-760, 916-932)
      - 16 bytes per stub (4 instructions): lis/ori/lwz/mtctr/bctr
      - Reserved2 field: 16 (vs 6 for x86_64, 12 for arm64)
      - Relocations: R_PPC_HA16, R_PPC_LO16
   
   c) Stub Helper (lines 800-830)
      - Common entry point for PLT resolution
      - Load _dyld_private address, call dyld_stub_binder
      - 32 bytes (larger than x86_64/arm64 variants)
      - Relocations: R_PPC_HA16, R_PPC_LO16

5. RELOCATION TYPE MAPPING
   - No direct Mach-O relocation conversion needed (code uses ELF format internally)
   - Just needs correct ELF relocation types for PPC:
     R_PPC_ADDR32, R_PPC_HA16, R_PPC_LO16, R_PPC_REL24, R_PPC_GLOB_DAT, R_PPC_JMP_SLOT
   
   (These should already be defined in tcc/elf.c; verify if TCC_TARGET_PPC added)

6. SYMBOL TABLE & DLL LOADING
   - Multiple locations access struct nlist_64 directly
   - Need conditional paths for 32-bit nlist struct
   
   Locations to update:
   - Line 1913: symlc->nsyms division
   - Line 2410: symtab allocation in DLL loading
   - Line 2462: symbol iteration in DLL loading

7. MACH-O HEADER INITIALIZATION
   - Magic number must be MH_MAGIC (0xfeedface) for PPC, not MH_MAGIC_64
   - CPU type: CPU_TYPE_POWERPC (18)
   - CPU subtype: CPU_SUBTYPE_POWERPC_ALL (0)
   
   (Magic already defined; conditionals needed at lines 1964-1970)

8. FAT BINARY PARSING
   - DLL loading from FAT binaries must recognize PPC slices
   - Location: lines 2376-2382
   - Check: CPU_TYPE_POWERPC + CPU_SUBTYPE_POWERPC_ALL

 PORTING STRATEGY (RECOMMENDED)
================================================================================

APPROACH: Monolithic File with Per-Arch #ifdefs

RATIONALE:
  - ~70% of code is shared (FAT parsing, symbol handling, DLL loading, export tries)
  - Splitting into per-arch files would duplicate thousands of lines
  - Conditional code is concentrated (instructions, load commands, struct sizes)
  - Single source of truth easier to maintain

STRUCTURE:
  1. Top of file: Define per-arch typedefs and helpers
  2. Constants: Platform guard + arch-specific CPU types
  3. Struct defs: 32-bit variants alongside 64-bit
  4. Helper funcs: Use typedefs to avoid repetition
  5. Core logic: Mostly unchanged, use typedefs
  6. Instruction blocks: #ifdef for each architecture
  7. Load commands: #ifdef for LC_MAIN vs LC_UNIXTHREAD
  8. DLL loading: Conditional magic/size checks

TOTAL CODE ADDED: ~480 lines
  - Struct definitions: ~50 lines
  - Typedefs and helpers: ~30 lines
  - #ifdef blocks and conditionals: ~400 lines
