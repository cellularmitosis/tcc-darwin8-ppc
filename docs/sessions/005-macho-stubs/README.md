# 005 — Mach-O integration stubs and first end-to-end execution

🎉 **Milestone:** `tcc -run` executes PPC code generated by our
backend on a real G3, returning the right exit code. First true
end-to-end win.

## Arrival state

After 004, tcc emitted correct PPC instructions but couldn't run
them. `tcc -run` was gated on `TCC_IS_NATIVE`, which in turn
required `tccmacho.c` (excluded since 003 because it's hardcoded
for x86_64/arm64).

## What was done

1. **`tcc/ppc-macho-stubs.c` added.** Minimal integration shims
   for the four Mach-O symbols `libtcc.c` references unconditionally:
   - `tcc_add_macos_sdkpath` — adds `/usr/lib` (Tiger has no
     `xcode-select`, no `.sdk` bundles).
   - `macho_load_dll`, `macho_load_tbd` — error stubs (no dynamic
     library loading yet).
   - `macho_output_file` — error stub (no Mach-O output yet).
   - `macho_tbd_soname` — returns NULL.
   Wired into the build via `Makefile` (replaces `tccmacho.c` in
   `ppc-osx_FILES`).

2. **`TCC_IS_NATIVE` enabled for PPC native** (uncommented the
   block in `tcc.h` from 004).

3. **`tccrun.c` runmain.o bypass for PPC.** The unmodified
   tccrun.c calls `tcc_add_support(s1, "runmain.o")` to load a
   wrapper that handles atexit/init_array/main-call sequencing.
   Loading `runmain.o` requires either:
   - Compiling `lib/runmain.c` with our tcc — blocked, our codegen
     can't yet handle init_array / atexit / extern symbols.
   - Loading a gcc-compiled `runmain.o` — blocked, our loader is
     ELF-only (Mach-O .o reading isn't implemented).
   So under `#ifdef TCC_TARGET_PPC` we skip the support load and
   call `main()` directly. Trade-off: no atexit, no static
   ctors/dtors. Acceptable for the early demo phase. Reverts when
   either of the unblockers above lands.

## Test record

```
$ ./tcc -B. -run /tmp/r42.c
exit=42

$ for v in 0 1 7 100 -1 65536 -65535 12345; do
    echo "int main(void){return $v;}" > /tmp/t.c
    ./tcc -B. -run /tmp/t.c; printf "  return %s -> exit %d\n" "$v" "$?"
  done
  return 0 -> exit 0
  return 1 -> exit 1
  return 7 -> exit 7
  return 100 -> exit 100
  return -1 -> exit 255       (Unix 8-bit truncation: -1 & 0xff = 255)
  return 65536 -> exit 0      (65536 & 0xff = 0)
  return -65535 -> exit 1     (-65535 = 0xffff0001 & 0xff = 1)
  return 12345 -> exit 57     (12345 & 0xff = 57)
```

All values correct after accounting for the Unix exit-code
truncation to low 8 bits.

Bonus: `int main(void) { return 1+2; }` returns 3. That's
constant-folding in tcc's parser working without our codegen — `1+2`
becomes literal `3` before reaching `load()`. So basic compile-time
arithmetic "works" already. Real runtime arithmetic on variables
needs `gen_opi`, which is still a stub.

## Limitations introduced (technical debt)

- **No atexit / no static init/fini arrays** in -run mode. From
  bypassing `runmain.o`. Will be fixed once we can either compile
  runmain.c with tcc or load Mach-O .o files.
- **No dynamic library loading** (printf, malloc, etc.). `macho_load_dll`
  is a stub. So programs that need libc are not yet runnable. We
  unblock this when tccmacho.c handles PPC (or we add a minimal
  Mach-O .dylib reader path).
- **No Mach-O output.** `tcc -c` emits ELF-format .o files that
  Tiger's `ld` can't link. Same unblock plan.

## Decisions

- **`ppc-macho-stubs.c` is a temporary scaffold,** not a permanent
  fixture. When tccmacho.c gains real PPC support (roadmap phase 3,
  estimated session ~020), this file is deleted and tccmacho.c
  picks up the slack.
- **Bypass under `#ifdef TCC_TARGET_PPC`** in tccrun.c, with a
  comment explaining the conditions for removing it. This is a
  surgical scope: doesn't affect any other target.

## Exit state

The G3 can compile and execute `int main(void) { return N; }` for
arbitrary N via `tcc -B. -run /tmp/file.c`. End-to-end pipeline
proven. We can iterate fast on codegen now: write a stub
implementation, test it on the G3, commit.

## Next session

Session 006 — start growing real codegen so we can compile useful
programs. Order of attack (from cheapest signal to most complex):

1. **Local variables and the assignment operator** (`store`, plus
   load for `VT_LOCAL`/`VT_LVAL`). Lets `int x = 7; return x;`
   work.
2. **Integer arithmetic** (`gen_opi` for + - * / and bitwise ops).
   Lets `int x = 1; return x + 2;` work.
3. **Branches** (`gjmp`, `gjmp_cond`, `gsym_addr`). Lets `if`,
   `while`, `for` work.
4. **Function calls** — Apple PPC ABI calling convention.

Each step gets a small C test under `tests/`, compiled and run via
`tcc -B. -run`, exit code checked.
